## 自顶向下的语法分析

### 语法分析器的功能

###### 语法分析器的作用

- 接受词法分析器提供的记号串
- 检查记号串是否能由源程序语言的文法产生
- 用易于理解的方式提示语法错误信息，并能从常见的错误中回复过来

![image-20211011105032554](https://cdn.jsdelivr.net/gh/KAQSICE/ImgHost/img/202110111050691.png)

###### 语法分析的例子

```java
{
    if (b == 0) 
        a = b;
    while (a != 1) {
        System.out.println(a);
        a = a - 1;
    }
}
```

###### 工作原理

语言的结构是用*上下文无关文法*描述的，因此语法分析器的工作本质就是按照文法的产生式，识别输入符号串是否为一个句子

语法分析器是从左向右的扫描输入字符串，每次读入一个符号，并判断，看是否能从文法的开始符号出发推导出这个输入串

或者，从概念上讲，就是要建立一棵与输入串匹配的*语法分析树*

语法分析器的分类:

- 通用的语法分析方法，用来分析任何文法，生成编译器时效率太低
- 编译器使用的语法分析方法，处理文法的一些子类
  - 自顶向下(建立分析树，即从句型到句子): LL文法，其分析器常用手工实现
  - 自底向上(建立分析树，即从句子到句型): LR文法，其分析器常利用自动生成工具构造

###  面临的问题

- 回溯
  - 自顶向下分析的主旨是，对任何输入串，穷尽一切可能的办法，从文法的开始符号出发，自顶向下的为输入串建立一棵语法树
  - 这种分析过程本质上是一种试探过程，是反复使用不同产生式谋求匹配输入串的过程，因而代价极高。
- 左递归导致的无限循环
  - 左递归问题导致分析过程陷入无限循环，因此必须消除文法的左递归性
  - 一个文法是左递归的，如果存在非终结符$P$，$P=^+>P\alpha$
- 虚假匹配
  - 即某些"成功"匹配是暂时性的

### LL(1)分析法

###### 消除直接左递归

使用如下公式消除直接左递归
$$
P\rightarrow P\alpha |\beta(\beta不以P开头)\\
\Downarrow\\
P\rightarrow\beta P'\\
P'\rightarrow\alpha P'|\epsilon
$$
得到的非直接左递归形式的文法与原文法等价

###### 预测分析表

预测分析表是一个$M[A,a]$形式的矩阵，其中A为非终结符，a是终结符或'#'

矩阵元素中存放一条关于A的产生式，指出当A面临输入a时应采用的候选

也可以存放一个出错标志，指出A根本不应该面临输入符号a
